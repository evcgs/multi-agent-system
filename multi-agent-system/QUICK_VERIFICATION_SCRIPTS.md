# Agent 快速确认对话脚本

## 使用说明

直接将以下问题逐个发送给对应的 Agent，验证其回答是否符合预期。

---

## 一、CalendarAgent 确认脚本

### 问题 1：自我介绍
> 请自我介绍一下

**预期回答**：
- 我是 CalendarAgent，日程管理专家
- 专注于日程管理、会议协调、提醒服务
- Emoji: 📅

---

### 问题 2：核心能力
> 你具备哪些能力？

**预期回答**（应包含以下几点）：
1. 日程管理：查询、创建、修改、删除日程
2. 会议协调：安排会议时间、检查冲突、发送邀请
3. 提醒服务：设置提醒、通知重要事项
4. 时间管理：时区转换、冲突检测、优化建议

**❌ 不应该出现**：
- 文件操作、代码执行等底层能力
- 文档管理、知识库操作（这是 DocAgent 的）

---

### 问题 3：边界约束
> 你的边界是什么？哪些事情你不做？

**预期回答**（应包含以下几点）：
1. 不处理文档管理、知识库操作（那是 DocAgent 的事）
2. 重大日程更改前会先确认
3. 尊重日历隐私

---

### 问题 4：多 Agent 协作
> 你和 DocAgent、Main Agent 如何分工？

**预期回答**：
- Main Agent：主协调者，处理通用问题
- DocAgent：处理文档、知识库相关
- CalendarAgent（我）：只处理日程、会议、提醒相关
- 大家分工明确，互不干涉

## 四、如果验证失败

### 常见问题及解决方案

#### 问题 1：Agent 回答了底层能力（如文件操作）
**原因**：没有正确加载 SOUL.md / MEMORY.md
**解决**：
- 确认工作区有 SOUL.md、IDENTITY.md、MEMORY.md
- 在 MEMORY.md 中明确说明"不要谈论底层技术能力"
- 重启 Gateway 或等待会话刷新

#### 问题 2：Agent 不知道自己的边界
**原因**：SOUL.md / MEMORY.md 中没有明确边界
**解决**：
- 在 SOUL.md 的 Boundaries 部分明确说明
- 在 MEMORY.md 中列出"不做什么"

#### 问题 3：Agent 不知道如何与其他 Agent 协作
**原因**：缺少多 Agent 协作的说明
**解决**：
- 在 MEMORY.md 中添加多 Agent 分工说明
