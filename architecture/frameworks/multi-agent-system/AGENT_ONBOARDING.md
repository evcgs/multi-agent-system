# Agent 初始化与确认流程（标准规范）

## 目的

确保每个新增的 Agent 都经过标准化的初始化、能力确认、分工边界明确，以及多 Agent 协调验证。

---

## 一、Agent 初始化检查清单

### 1.1 基础配置检查
- [ ] **IDENTITY.md** - Agent 身份定义
  - 名称（Name）
  - 角色定位（Creature）
  - 风格调性（Vibe）
  - 标志性 Emoji
- [ ] **SOUL.md** - Agent 灵魂与行为准则
  - 核心真理（Core Truths）
  - 边界约束（Boundaries）
  - 风格调性（Vibe）
- [ ] **MEMORY.md** - 长期记忆（能力声明）
  - 明确的核心能力列表
  - 禁用的能力/话题
  - 与用户交互的约定
  - 包含「会话初始化强制流程」和「信息完整性校验准则」
- [ ] **TOOLS.md** - 工具与凭证配置
  - 可用的工具列表
  - 凭证信息（脱敏存储）
  - 配置路径
  - 包含 global-awareness 全局感知技能
- [ ] **BOOTSTRAP.md** - 初始化完成标记

### 1.2 技术配置检查
- [ ] **飞书机器人权限**
  - `im:message:send` 或 `im:message:send_as_bot`
  - 其他业务相关权限
- [ ] **OpenClaw 配置**
  - `openclaw.json` 中的 agents.list 配置
  - bindings 配置正确
  - 账号配置正确

## 三、初始化流程步骤

### 步骤 1：创建 Agent 工作区
```
~/.openclaw/agents/{agent_id}/workspace/
```

### 步骤 2：生成配置文件
- IDENTITY.md
- SOUL.md
- MEMORY.md
- TOOLS.md
- BOOTSTRAP.md

### 步骤 3：配置 OpenClaw
- 更新 openclaw.json
- 添加 agent 到 agents.list
- 添加 binding

### 步骤 4：开通飞书权限
- 消息发送权限
- 业务相关权限

### 步骤 5：能力确认对话
- 与 Agent 进行初始化对话
- 让 Agent 自我介绍
- 确认能力清单

### 步骤 6：生成确认报告
- 填写能力确认模板
- 签字确认

### 步骤 7：多 Agent 协作测试
- 测试与其他 Agent 的交互
- 验证分工边界

## 五、快速开始：新增 Agent

1. 复制此模板到 `multi-agent-system/agents/{agent_id}-ONBOARDING.md`
2. 按照检查清单逐步执行
3. 完成后将此文件移动到 `multi-agent-system/archives/`
