# 多Agent协同系统 - 架构设计

## 1. 设计原则

### 1.1 核心原则
- **结果导向**：人类只定义可量化、可验证的最终目标
- **不干涉过程**：放弃对AI的微操，不教AI「怎么做事」
- **风险可控内授予最高权限**：先锁死风险边界，再给AI最大操作空间

### 1.2 架构原则
- **职责单一**：每个Agent只负责一个明确的能力域
- **高内聚低耦合**：Agent内部高内聚，Agent之间低耦合
- **可扩展性**：易于添加新的能力域和Agent
- **容错性**：单个Agent失败不影响整个系统

## 3. 主Agent详细设计

### 3.1 核心模块

#### 3.1.1 意图理解模块 (Intent Parser)
**职责**：理解用户的需求和意图

**输入**：用户的自然语言输入

**输出**：结构化的意图描述
- 意图类型（查询/操作/创作/协调等）
- 涉及的能力域
- 关键参数和约束
- 优先级和时间要求

**示例**：
```json
{
  "intent": "create_document",
  "domains": ["doc"],
  "parameters": {
    "title": "项目计划",
    "content": "..."
  },
  "priority": "normal"
}
```

#### 3.1.2 能力路由模块 (Capability Router)
**职责**：根据意图判断需要调用哪些能力域

**输入**：结构化的意图描述

**输出**：路由决策
- 需要调用的Agent列表
- 调用顺序（串行/并行）
- 数据传递关系

**路由规则**：
- 单一意图 → 单一Agent
- 复合意图 → 多个Agent协同
- 跨域意图 → 主Agent协调

#### 3.1.3 任务分发模块 (Task Dispatcher)
**职责**：将任务分发给对应的子Agent

**输入**：路由决策 + 原始意图

**输出**：子Agent的执行结果

**分发方式**：
- 通过 `sessions_spawn` 创建子Agent会话
- 通过 `sessions_send` 发送任务
- 等待/收集子Agent的返回结果

#### 3.1.4 结果汇总模块 (Result Aggregator)
**职责**：整合多个子Agent的输出

**输入**：多个子Agent的执行结果

**输出**：统一的最终答案

**汇总策略**：
- 简单拼接
- 逻辑整合
- 冲突解决
- 格式统一

#### 3.1.5 上下文管理模块 (Context Manager)
**职责**：维护全局上下文

**功能**：
- 对话历史管理
- 任务状态跟踪
- 数据缓存和共享
- 会话连续性维护

## 5. 协作流程

### 5.1 单一Agent流程

```
用户输入
    ↓
主Agent - 意图理解
    ↓
主Agent - 能力路由（确定需要Agent X）
    ↓
主Agent - 任务分发（调用Agent X）
    ↓
Agent X - 执行任务
    ↓
Agent X - 返回结果
    ↓
主Agent - 结果汇总
    ↓
返回给用户
```

### 5.2 多Agent协同流程

```
用户输入
    ↓
主Agent - 意图理解
    ↓
主Agent - 能力路由（确定需要Agent A + Agent B）
    ↓
主Agent - 任务分发
    ├─→ 调用Agent A（并行）
    └─→ 调用Agent B（并行）
    ↓
Agent A - 执行任务
Agent B - 执行任务
    ↓
Agent A - 返回结果
Agent B - 返回结果
    ↓
主Agent - 结果汇总（整合A + B的结果）
    ↓
返回给用户
```

## 6. 配置方案

### 6.1 OpenClaw 配置结构

```yaml
channels:
  feishu:
    # 主Agent（默认账户）
    enabled: true
    appId: "cli_main_agent"
    appSecret: "xxx"
    # 子Agent账户
    accounts:
      doc-agent:
        name: "DocAgent"
        enabled: true
        appId: "cli_doc_agent"
        appSecret: "yyy"
      calendar-agent:
        name: "CalendarAgent"
        enabled: true
        appId: "cli_calendar_agent"
        appSecret: "zzz"
      data-agent:
        name: "DataAgent"
        enabled: true
        appId: "cli_data_agent"
        appSecret: "aaa"
```

### 6.2 Agent绑定配置

```yaml
bindings:
  - agentId: "main"
    match:
      channel: "feishu"
      accountId: "default"
  - agentId: "doc-agent"
    match:
      channel: "feishu"
      accountId: "doc-agent"
  - agentId: "calendar-agent"
    match:
      channel: "feishu"
      accountId: "calendar-agent"
```
